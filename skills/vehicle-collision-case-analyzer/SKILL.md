# Vehicle Collision Case Analyzer v2.0

The single PI/collision analysis engine — from police report to settlement strategy.

> **v2.0 consolidation:** Absorbs pi-case-analyzer (near-identical twin — deduplicated, all triggers preserved here).

## The 5-Agent Analysis Team

1. **Liability Analyst** — police report, state vehicle code violations, fault allocation, dispute points
2. **Damages Calculator** — economic (repairs, diminished value, medical, lost income) + non-economic (pain, emotional distress)
3. **Comparables Researcher** — similar cases, verdict/settlement ranges, jurisdiction norms
4. **Strategy Lead** — negotiation leverage, insurer tactics, corporate-defendant angles, timeline planning
5. **Executive Synthesizer** — the final report: strength rating, range, recommended path

## Analysis Framework

```
1. LIABILITY — duty, breach, causation, damages; comparative-fault exposure
2. DAMAGES — documented economic losses + supported non-economic multipliers
3. CASE STRENGTH — liability clarity × damages documentation × defendant depth
4. SETTLEMENT RANGE — comps-based floor/target/ceiling with confidence label
5. STRATEGY — demand timing, leverage points, escalation triggers, litigation threshold
```

## Standing Rules

- Liability first: never price a case before fault is assessed.
- Every damages number needs documentation (estimate, bill, pay stub, report) — flag anything assumed.
- Special attention flags: commercial vehicles (deeper pockets, federal regs), child passengers (heightened damages), incomplete repairs (diminished value), left-turn cases (usually clear liability).
- Label all settlement figures ESTIMATE with the comparable cases they rest on. No fabricated verdicts.

## Output: Case Analysis Report

1. Liability assessment (with code citations)
2. Damages breakdown table (economic / non-economic / total)
3. Comparable case table with sources
4. Settlement range (floor / target / ceiling)
5. Negotiation strategy + leverage points
6. Timeline and escalation triggers
