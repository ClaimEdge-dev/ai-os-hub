# Policy Analysis Framework

## 4-Type Coverage Classification

### Type 1: Confirmed Coverage
**Definition:** Coverage is explicitly granted by policy language.
**Indicators:**
- Policy names peril as covered
- No applicable exclusion
- Endorsement confirms coverage
**Action:** Document and demand payment

### Type 2: Likely Coverage
**Definition:** Coverage is strongly implied but requires argument.
**Indicators:**
- Ambiguous policy language
- Exclusion may not apply
- Case law supports coverage
**Action:** Develop argument, cite authority, demand coverage

### Type 3: Disputed Coverage
**Definition:** Carrier disputes coverage; requires formal challenge.
**Indicators:**
- Carrier cites exclusion
- Policy language is unclear
- Factual dispute exists
**Action:** Challenge exclusion, file IDOI complaint, prepare for appraisal

### Type 4: Excluded (Challengeable)
**Definition:** Coverage appears excluded but challenge exists.
**Indicators:**
- Clear exclusion language
- Statutory override possible
- Public policy argument available
**Action:** Challenge exclusion with statutory/case law arguments

## Coverage Verification Checklist

### Pre-Analysis
- [ ] Policy declarations page obtained
- [ ] All endorsements identified
- [ ] Policy period confirmed (covers loss date)
- [ ] Named insured confirmed
- [ ] Property address matches claim

### Coverage Types
- [ ] Coverage A (Dwelling) - amount and type (RCV/ACV)
- [ ] Coverage B (Other Structures) - percentage of A
- [ ] Coverage C (Personal Property) - amount and limitations
- [ ] Coverage D (Loss of Use) - amount and duration
- [ ] Ordinance or Law - percentage and triggers
- [ ] Additional endorsements identified

### Exclusions Analysis
- [ ] All applicable exclusions listed
- [ ] Each exclusion classified (valid/ambiguous/challengeable)
- [ ] Challenge strategy identified for each
- [ ] Supporting statutes cited
- [ ] Case law supporting arguments identified

### Code Upgrade Analysis
- [ ] Current adopted code identified
- [ ] Code upgrade coverage amount confirmed
- [ ] Specific code triggers identified
- [ ] Local amendments researched
- [ ] Code compliance letter requested from building department

### Carrier-Specific Factors
- [ ] Carrier's standard positions researched
- [ ] Prior similar claims identified
- [ ] Carrier's financial condition noted
- [ ] Reinsurance layer identified (if applicable)
